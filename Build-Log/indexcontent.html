Build Instruction:Part Crib Project

Introduction

For my Hardware project, I decided to work on the Parts Crib database. With all of the knowledge I have acquired in my previous courses, I believe I have the skills to properly design and program a piece of hardware that will work in conjunction with a smartphone/tablet application. The main purpose is to bring a more efficient and advanced experience between students and employees. Right now the parts crib is using paper and pen to sign in/out equipment along with leaving your student ID. With our hardware we plan to modernize the experience by scanning in each student’s library barcode to identify students and scanning QR codes to identify parts and link the rented equipment with the student barcode ID in our network database. The database will also collectively produce data that can be used to help identify higher demands for certain items and also help track lost inventory. My project team mates are Masoud Rahguzar, Divesh Oree and Rafil Yashooa.

System Diagram

The webcam will be used to scan ID barcodes and item QR codes to register the parts with students. The codes will then be processed by the raspberry pi. Once everything is processed the Pi will upload the info to the database which can then be accessed through a mobile app for a better visual experience.

Mechanical Assembly

The assembly of my project is quite simple. The webcam connects to the raspberry pi through the USB connector. The PCB is then connected to the 40 pin connector on top of the raspberry pi. Also connect the HDMI to a monitor for viewing and connected a power source to the Pi.

PCB / Soldering

The PCB contains resistors, capacitors, transistors, an LED, 2 four pin headers and a 5 pin header. All of these components were soldered together using non led based solder and took about an hour to properly solder everything without any errors. It is best to take your time and fill the entire circular area on the board with enough but not too much solder.

Software Setup

Start by assembling the raspberry pi into its casing. Once all assembled make sure to have your HDMI cable connected first along with a keyboard and mouse. Before powering up your Raspberry Pi make sure your SD card has the appropriate operating system. You can visit Raspberrypi.org to download the free operating system and the instructions on how to install on an SD card. Once that is finished, insert the SD card into the Raspberry Pi and connect the micro USB power adaptor. The unit will take a few minutes for its first time boot. Once you have powered up make sure to open up the terminal and run: 

sudo apt-get update
sudo apt-get upgrade
This will ensure that you have updated to the latest firmware and drivers.Next download Necessary software. Start by installing the webcam driver by running: 
sudo apt-get install fswebcam
Next you’ll need to install python, zbar library and pillow by running: 
sudo apt-get install python-dev
sudo apt-get install python-pip
sudo apt-get install python-pip
sudo apt-get install python-httplib2
Once all that is installed and downloaded you’ll need to download the zbar library from the official Github account and export the files into a single folder. Once downloaded run: 
python setup.py install –user
This will install the appropriate files and will be done automatically once the command runs. Then download the code listed below and run: 
gcc –Wall –o executable name c_file.c – lwiringPi"
Once that code is compiled run the executable name that you chose for the program and youll be able to scan barcodes and view them in a file called student_numbers

Unit Testing

The unit testing is always the first thing that should be done in order to ensure the reliability of the hardware. The PCB was the main board we needed to test since it was the only one that had been soldered by us. Most of our hardware was pre-built and needed to just be connected together. I was able to successfully test my PCB in the prototype lab with Vlad and all the components were working fine with no shorts or fixes required.

Production Testing

Production testing can be done by running each individual code by itself to ensure each component is working. The webcam can be tested by taking screen shots and checking the output file to see if the webcam is able to see anything.
